# Ubiquitous Language

> 来源：Eric Evans《领域驱动设计》第 2 章 (2003) / Matt Pocock
> 触发词：统一语言, ubiquitous language, 术语表, 对术语

## 执行指令

扫描当前对话，提取领域相关术语，输出标准化的术语表。

## 流程

1. **扫描对话** — 提取领域相关的名词、动词、概念
2. **识别问题** — 同一词表不同概念（歧义）、不同词表同一概念（同义词）、模糊或重载术语
3. **提出规范术语** — 为每个概念选定唯一的规范名称，列出应避免的别名
4. **写入 CONTEXT.md** — 用下方格式更新项目的 CONTEXT.md
5. **输出摘要**

## CONTEXT.md 格式

```md
# {上下文名称}
{一两句话描述}

## 术语

**Term**: 一句话定义。定义它是什么，不是它做什么。
_Avoid_: 别名1, 别名2

## 关系

- 一个 **A** 产生一个或多个 **B**

## 示例对话

> **Dev:** "..."
> **Domain expert:** "..."

## 待澄清

- "X" 被用于指代 Y 和 Z — 需区分
```

## 规则

- **有主见** — 存在多词指同一概念时，选定最佳者，其余列为别名
- **明确标出冲突** — 术语有歧义时标在"待澄清"中
- **只包含领域术语** — 跳过通用编程概念（除非有特定领域含义）
- **定义简洁** — 一句话以内
- **写示例对话** — 3-5 轮 Dev 与 Domain Expert 对话，展示术语如何精确交互
- **重复执行时** — 读取已有 CONTEXT.md，只增量更新新术语
